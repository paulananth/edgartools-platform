**Type:** Missing configuration  
**Evidence:** `printenv | grep -i neo4j` returns nothing. `verify-graph` exits with `"error": "NEO4J_URI/NEO4J_USER/NEO4J_PASSWORD not configured"`. Dashboard overview will show Neo4j as "not configured" (orange warning).  
**Impact:** Anyone running the dashboard or CLI commands locally sees Neo4j as absent — even though it's live.

  --query SecretString --output text)
# Then run the dashboard:
