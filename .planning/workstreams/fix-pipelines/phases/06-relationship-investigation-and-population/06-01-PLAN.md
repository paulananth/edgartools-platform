<objective>
Implement the CIK-range batched-read strategy for INSTITUTIONAL_HOLDS (D-03) inside
`_derive_institutional_holds` so the 06-03 bounded `load_history` run can derive
INSTITUTIONAL_HOLDS at scale without the OOM risk TODOS.md flagged. This is a deliberate
user override (D-03): build batching now, not deferred, because D-02 puts a real load in
this phase and makes the OOM risk live rather than hypothetical.

Purpose: `SilverDatabase.fetch()` returns `list[dict]` — all rows in memory at once.
`sec_thirteenf_holding` is the largest silver table (large fund managers report tens of
thousands of positions per quarter); a full scan can be 10M+ rows in one list and exceed
ECS task memory. Batching by CIK range bounds each read.
Output: A batched, parameterized `_derive_institutional_holds` with regression tests
proving batched output equals single-query output.
</objective>
