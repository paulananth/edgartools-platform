<objective>
Prove node-derivation idempotency across all 6 MDM entity types (GVER-03, node side).

`tests/mdm/test_pipeline_relationships.py` already proves relationship-derivation idempotency
(`test_relationship_derivation_is_idempotent`, `test_all_relationship_types_idempotent`) against
a real SQLAlchemy session — the pattern that surfaced the plateau-fix regression a mock could
not have caught (D-04). GVER-03 requires the same repeatable, automated proof for the 6 node
types, which today has no coverage. This plan extends the proven real-DB pattern to the node
side: run entity resolution / seeding twice against unchanged inputs and assert zero net-new
active entities per type.

Purpose: GVER-03 spans "all 6 node types and 11 relationship types." The relationship half is
covered; this plan closes the node half with a committed regression test rather than the ad-hoc
live sync-graph observation, which D-05 explicitly says does not satisfy GVER-03 on its own.
Output: A real-DB node-idempotency test covering company, adviser, person, security, fund, and
audit_firm.
</objective>
