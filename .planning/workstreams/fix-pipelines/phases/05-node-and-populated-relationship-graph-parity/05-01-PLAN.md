<objective>
Close the NODE-06 view gap and prove graph-sync idempotency (GVER-03, sync side).

`render_graph_tables()` in `edgar_warehouse/mdm/snowflake_graph.py` emits five per-type node
views (company, person, security, adviser, fund) but no `GRAPH_NODE_AUDITFIRM` view — even
though `GRAPH_NODE_AUDITFIRM` is already declared in `NODE_TABLES` and `audit_firm` is already
in `ALLOWED_ENTITY_TYPES`. This is exactly the missing-view gap D-02 identifies: NODE-06 has
nothing to verify until the view exists. This plan adds that view following the existing
per-type pattern, and adds the graph-sync (full-rebuild) idempotency regression test that
GVER-03 requires on the sync side.

Purpose: NODE-06's per-type parity check (added in plan 03) needs the `GRAPH_NODE_AUDITFIRM`
view to exist; and GVER-03 requires a committed regression test that repeated `sync-graph`
against unchanged MDM state is a stable no-op, not a one-time manual observation (D-05).
Output: An emitted `GRAPH_NODE_AUDITFIRM` view + a graph-sync idempotency test.
</objective>
