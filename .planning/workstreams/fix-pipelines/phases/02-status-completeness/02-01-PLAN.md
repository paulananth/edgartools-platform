<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → status.sh | Script is invoked by a human operator with AWS credentials; no untrusted input enters the script beyond the optional `--env`, `--region`, `--profile`, and filter positional argument |
| AWS CLI output → embedded Python | JSON from `aws stepfunctions get-execution-history` is piped to embedded Python; the JSON is trusted (AWS account-controlled) but must be validated for shape |
| env var STAGES → embedded Python | Stage names are passed via the `STAGES` environment variable; sourced from the hardcoded MACHINES array, never from user input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering / Injection | `STAGES` env var → `os.environ.get('STAGES','').split()` in embedded Python | Accept | `STAGES` is set by the shell from the hardcoded MACHINES array (`"$stages_str"`), never from user-supplied arguments. No injection path exists in practice. If the script is ever extended to accept stage names from a flag, this must be re-evaluated. |
| T-02-02 | Information Disclosure | `assert len(active) <= 1, f"one-▶ invariant violation: {active}"` — asserts message contains stage names | Accept | Stage names (e.g., `SeedUniverse`, `MdmRun`) are public — they appear in open-source state machine definitions and the CLAUDE.md architecture docs. No PII or secrets in stage names. Exposure is limited to operators who already have AWS credentials to run the script. |
| T-02-03 | Tampering | AWS CLI output (`get-execution-history`) processed by embedded Python | Accept | The script consumes AWS API output from the caller's own account. The caller must have valid AWS credentials with Step Functions read permissions. No cross-account or external trust boundary. Malformed JSON from a corrupted or spoofed response would cause a Python exception — acceptable for an operator diagnostic script (`set -euo pipefail` ensures the script fails loudly). |
| T-02-04 | Elevation of Privilege | `bash -n scripts/ops/status.sh` on a modified file | Accept | The script is read-only diagnostic. It makes no writes to AWS state, no IAM changes, no file mutations. Execution privilege is limited to AWS read APIs. |
| T-02-05 | Denial of Service | Operator runs status.sh in a tight loop against AWS APIs | Accept | Status.sh is an operator-invoked diagnostic; it is not on an automated schedule. AWS Step Functions API rate limits (10 req/s for list-executions) apply; the script fetches at most 5 executions × 5 machines = 25 calls per invocation. No throttling protection needed for a human-invoked tool. |

**Verdict:** No high-severity threats. All threats accepted with documented rationale. No mitigations required to proceed.
