## Requirements — Final Status

### Failure Surfacing

- [x] **OBS-01**: Any stage failure within `bootstrap_phased` causes the execution to reach
  FAILED state — the execution must never reach SUCCEEDED when a stage errored or produced
  partial output
  **Outcome:** VALIDATED — `ToleratedFailurePercentage: 0` deployed; test script created.
  Note: live runtime re-test was aborted due to S3 race condition in test script; definition-level verification passed.

- [x] **OBS-02**: The same hard-fail behavior applies to all sibling state machines:
  `silver-mdm-gold`, `gold-refresh`, `mdm-gold`, `ownership-mdm-gold`
  **Outcome:** VALIDATED — same deploy script edit applies to all 5 state machine definitions.

### Status Completeness

- [x] **OBS-03**: `status.sh` shows a complete stage-level breakdown for all 5 registered
  state machines — no stages are silently missing from the display
  **Outcome:** VALIDATED — `ownership-mdm-gold` added; all 5 machines covered.

- [x] **OBS-04**: `status.sh` shows which stage is actively executing during a running
  pipeline — not just "RUNNING" at the top level
  **Outcome:** VALIDATED — `▶` marker wiring verified; one-active-stage assertion added.

### Failure Notifications

- [x] **OBS-05**: When any pipeline execution reaches FAILED state, an SNS email notification
  is sent to a configured subscriber (email address configurable via Terraform variable)
  **Outcome:** VALIDATED — live end-to-end test: mdm-gold FAILED → email received within 60s.

- [x] **OBS-06**: The SNS notification body identifies: pipeline name, execution ARN, failed
  stage name, and a CloudWatch Logs deep-link for the failing ECS task
  **Outcome:** PARTIALLY SATISFIED — "failed stage name" descoped (D-06): not available in
  EventBridge event schema without Lambda (forbidden by D-01). Body contains: full
  stateMachineArn (pipeline identity), execution ARN, failure epoch timestamp, Step Functions
  console link, static CloudWatch log group URL. Accepted as tech debt.

### SEC Rate Limiting

- [x] **SEC-RL-01**: All direct SEC EDGAR HTTP calls through `download_sec_bytes()` are
  subject to an in-process rate limiter of 9 requests/second
  **Outcome:** VALIDATED — `_SEC_RATE_LIMITER` singleton in sec_client.py; unit test confirms
  single token per logical request regardless of retry count; all 4 tests pass.

- [x] **SEC-RL-02**: The recommended `BOOTSTRAP_BATCH_CONCURRENCY` range (2–5) is documented
  in `CLAUDE.md`, with current default of 3 noted as compliant
  **Outcome:** VALIDATED — Key invariants bullet added to CLAUDE.md with range, rationale,
  and explicit default compliance statement.
