## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator -> Snowflake account | Human operator grants app privileges and application roles. |
| Native App -> consumer data | Application receives database/schema/table/view permissions to read graph inputs and write algorithm outputs. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Elevation of Privilege | Snowflake grants | mitigate | Runbook must distinguish documented examples from platform grants and require review before broad privileges are used. |
| T-01-02 | Information Disclosure | Event sharing and app logs | mitigate | Runbook must require operator acknowledgement of event sharing and avoid storing secrets in planning docs. |
| T-01-03 | Denial of Service | Compute pools and warehouse | mitigate | Runbook must document compute-pool auto-suspend defaults and live status checks before implementation phases depend on them. |
</threat_model>
