# Phase 04 - UI Design Contract

> Visual and interaction contract for the Phase 4 MDM graph dashboard migration.
> Generated by `gsd-ui-phase`; downstream planning must treat this as the UI source of truth.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | Streamlit native primitives |
| Icon library | none |
| Font | Streamlit default sans-serif |

Phase 4 must preserve the current local Streamlit dashboard shape. Do not introduce a React app, custom component framework, decorative hero, marketing layout, card-heavy redesign, custom charting library, or browser-side state model. The dashboard is an operational review tool for repeat use.

Use only Streamlit-native controls already present in the dashboard unless a missing requirement cannot be represented with them:
- `st.sidebar.radio` for section navigation.
- `st.sidebar.selectbox` for row limit.
- `st.sidebar.button` for manual refresh.
- `st.selectbox` for entity type and relationship type filters.
- `st.metric` for compact summary counts.
- `st.dataframe` for diagnostic tables.
- `st.info`, `st.warning`, `st.error`, and `st.success` for states.
- `st.caption` for timestamps, bounded-sample notices, and non-primary context.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps inside captions and short labels |
| sm | 8px | Table/control adjacency, compact status grouping |
| md | 16px | Default spacing between controls and tables |
| lg | 24px | Between major diagnostic sections |
| xl | 32px | Page title to first dashboard band when attention states are absent |
| 2xl | 48px | Maximum separation before secondary diagnostic groups |
| 3xl | 64px | Not used in this dashboard |

Exceptions: Streamlit's built-in sidebar, metric, dataframe, and alert component padding may remain at framework defaults. Do not add custom CSS merely to force exact pixel spacing.

Layout constraints:
- Keep the existing wide page layout.
- Do not nest UI cards inside other cards.
- Do not use floating page-section cards.
- Tables must use full available width with stable bounded row counts.
- Text must not overlap controls or table content at desktop or narrow widths.

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 500 | 1.4 |
| Heading | 24px | 600 | 1.25 |
| Display | 32px | 700 | 1.2 |

Use Streamlit semantic primitives for typography:
- Page title: `st.title`.
- Section heading: `st.subheader`.
- Explanatory metadata: `st.caption`.
- Attention labels: built-in alert components, not oversized headings.

Do not scale font size with viewport width. Do not use negative letter spacing. Keep table column names short enough to fit without truncating the data issue being diagnosed.

---

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FFFFFF | Main page background and table surfaces |
| Secondary (30%) | #F6F8FA | Sidebar, subtle section separation where Streamlit supports it |
| Accent (10%) | #2563EB | Manual refresh focus state and informational emphasis only |
| Success | #16A34A | Healthy checks and no-issue state |
| Warning | #D97706 | Coverage warnings, stale/missing graph data |
| Destructive | #DC2626 | Blocking errors only; no destructive actions exist |

Accent reserved for: informational emphasis, focus affordance, and links to non-mutating runbook commands. Do not use accent color for every interactive element.

Use Streamlit built-in semantic states rather than custom color overrides where possible:
- `st.success` for "No blocking failures or coverage warnings."
- `st.warning` for mismatch, missing graph data, stale status, and Native App readiness failures that leave MDM review available.
- `st.error` for MDM configuration, MDM connection, or MDM permission blockers.
- `st.info` for empty filtered results and offline guidance.

Do not add gradients, decorative background blobs, hero imagery, or one-note blue/purple visual themes.

## Navigation Contract

Keep these section labels and ordering:

1. `Overview`
2. `MDM Overview`
3. `Neo4j Overview`
4. `Mismatch Diagnostics`

`Neo4j Overview` remains the route label for continuity, but page captions and failure copy must make clear that the graph target is Snowflake-hosted Neo4j Graph Analytics, not external Aura/Bolt connectivity.

The sidebar must keep:
- Title: `EdgarTools MDM`
- Caption updated away from external Neo4j wording, for example `Read-only MDM and Snowflake-hosted graph status`.
- `Section` radio.
- `Row limit` selectbox with options `25`, `50`, `100`, `250`, default `50`.
- `Refresh metrics` button.

No sidebar control may run sync, repair, migrate, load, derive, write, grant, activate, or deploy actions.

## Interaction Contract

### Refresh

Primary CTA: `Refresh metrics`

Behavior:
- Clears cached read-only dashboard payloads.
- Reruns read-only helper reads only.
- Does not run `sync-graph`, `verify-graph`, Native App activation, grants, migrations, or AWS E2E.
- After refresh, pages show visible last-checked timestamps for MDM and Snowflake graph payloads when available.

### Filters

Filters must remain single-select:
- `Entity type`: default `All`; options come from the MDM registry and current metrics.
- `Relationship type`: default `All`; options come from the MDM registry and current metrics.

Filtered empty state:
- Heading: `No rows match the current filters.`
- Body: `Adjust the selected type or row limit, then review the table again.`

### Data Freshness

Every loaded payload should expose a timestamp rendered as:
- `MDM metrics last refreshed: <timestamp>`
- `Snowflake graph metrics last refreshed: <timestamp>`

If a timestamp is missing, do not fabricate one. Show the table or status without timestamp and let tests cover the missing field.

## View Contracts

### Overview

Purpose: triage first, then counts.

Required order:
1. Page title.
2. Read-only hosted graph caption.
3. Blocking failures and coverage warnings.
4. Compact metric row.
5. Last-refreshed captions.

Summary metrics:
- `MDM entities`
- `MDM relationships`
- `Snowflake graph nodes`
- `Snowflake graph edges`
- `Pending sync`

The overview must not always show Native App `GRAPH_INFO`, `BFS`, `WCC`, or compute-pool detail when those checks are healthy.

### MDM Overview

Purpose: inspect source-of-truth MDM state.

Required elements:
- `Entity type` filter.
- `Relationship type` filter.
- MDM last-refreshed caption.
- Entity counts table.
- Relationship counts table.

The page remains usable when Snowflake graph data is unavailable.

### Neo4j Overview

Purpose: inspect Snowflake-hosted Neo4j Graph Analytics counts and parity summary.

Required elements:
- Caption or state copy that says the graph is hosted in Snowflake.
- `Entity type` filter.
- `Relationship type` filter.
- Snowflake graph last-refreshed caption when available.
- Entity comparison table.
- Relationship comparison table.

Preferred table columns:
- Entity comparison: `Entity Type`, `MDM Active`, `Snowflake Graph Nodes`, `MDM Minus Graph`, `Graph Minus MDM`, `Status`.
- Relationship comparison: `Relationship Type`, `MDM Active`, `Snowflake Graph Edges`, `MDM Minus Graph`, `Graph Minus MDM`, `Status`.

When graph data is unavailable, show secret-safe guidance and keep `MDM Overview` available.

### Mismatch Diagnostics

Purpose: one-stop investigation surface for data issues.

Required elements:
- `Entity type` filter.
- `Relationship type` filter.
- Bounded-sample caption: `Samples are bounded diagnostics, not exhaustive diffs.`
- Relationship parity table by type.
- Missing graph nodes sample table.
- Extra graph nodes sample table.
- Missing graph edges sample table.
- Extra graph edges sample table.
- Missing graph edge endpoints sample table.

Required sample columns:
- Missing graph nodes: `Entity Type`, `Node ID`.
- Extra graph nodes: `Entity Type`, `Node ID`.
- Missing graph edges: `Relationship Type`, `Edge ID`.
- Extra graph edges: `Relationship Type`, `Edge ID`.
- Missing edge endpoints: `Relationship Type`, `Edge ID`, `Source Node ID`, `Target Node ID`, `Missing Source Node`, `Missing Target Node`, `Direction`.

Direction should be rendered as `source -> target` when both sides are known. Use text, not icons, so table rows remain copyable.

### Native App Failure Detail

Purpose: explain hosted graph readiness failures without making healthy proof noisy.

Display only when a Native App prerequisite or smoke check fails or is unavailable.

Required failure detail fields:
- `Check`
- `Status`
- `Detail`
- `Remediation`

Checks that may appear:
- Native App installation.
- App role grants.
- Database role grants.
- Schema/table privileges.
- Compute pool selector.
- Graph schema sample access.
- `GRAPH_INFO`.
- `BFS`.
- `WCC`.

Healthy `GRAPH_INFO`, `BFS`, `WCC`, and compute-pool values may be omitted from the main pages. If shown in an expanded/detail area, they must stay secondary to mismatch diagnostics.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | `Refresh metrics` |
| Empty state heading | `No rows match the current filters.` |
| Empty state body | `Adjust the selected type or row limit, then review the table again.` |
| Bounded samples | `Samples are bounded diagnostics, not exhaustive diffs.` |
| MDM missing config | `MDM configuration is required. Set \`MDM_DATABASE_URL\`, then restart the dashboard.` |
| MDM unavailable | `MDM database unavailable. Check \`MDM_DATABASE_URL\`, confirm the database is reachable, and restart the dashboard.` |
| MDM permission denied | `MDM database permission denied. Confirm the configured database user can run read-only SELECT queries.` |
| Hosted graph unavailable | `Snowflake graph metrics unavailable. MDM overview remains available.` |
| Hosted graph permission denied | `Snowflake graph permission denied. Confirm the configured Snowflake role can run read-only graph diagnostics.` |
| Native App unavailable | `Snowflake Native App check failed. Run \`edgar-warehouse mdm verify-graph\` for the acceptance gate and review the remediation below.` |
| Destructive confirmation | Not applicable - the dashboard has no destructive actions. |

Forbidden copy:
- Do not tell operators to configure `NEO4J_URI`, `NEO4J_USER`, `NEO4J_PASSWORD`, `NEO4J_DATABASE`, or `NEO4J_SECRET_JSON` for Phase 4 dashboard success.
- Do not mention external Bolt, Aura, or read-only `MATCH` as the active dashboard path.
- Do not print DSNs, passwords, tokens, hostnames from connection strings, stack traces, raw connector errors, or raw Snowflake exception payloads.
- Do not add visible instructions describing keyboard shortcuts, styling, or how the UI is built.

Allowed runbook references:
- `edgar-warehouse mdm verify-graph`
- `bash infra/scripts/run-aws-mdm-e2e.sh --env dev --aws-profile sec_platform_deployer --snow-connection snowconn --snowflake-database EDGARTOOLS_DEV`
- `infra/snowflake/sql/neo4j_graph_analytics_app_grants.sql`

Runbook references must remain documentation text only, not dashboard controls.

## Read-Only Safety Contract

The dashboard and helpers must not expose or call:
- `sync-graph`
- `derive-relationships`
- `load-relationships`
- `migrate`
- `seed-universe`
- repair, accept, reject, edit, delete, merge, quarantine, grant, activate, deploy, or AWS execution controls

The Streamlit file must remain a renderer over read-only helper payloads:
- No raw SQL in `examples/mdm_graph_dashboard/streamlit_app.py`.
- No raw Cypher in `examples/mdm_graph_dashboard/streamlit_app.py`.
- No subprocess calls from the dashboard.
- No stdout parsing.
- No credential entry fields.

---

## Responsive Behavior

Target: local operator browser sessions on laptop and desktop; narrow browser windows must remain usable.

Rules:
- Keep the sidebar controls stable and short.
- Data tables can scroll horizontally through Streamlit's dataframe component.
- Long IDs must appear in table cells without overlapping adjacent UI.
- Alert copy must wrap naturally; do not put long remediation commands inside button labels or metric labels.
- Metrics may wrap to multiple rows on narrow widths; they must not obscure the attention section or filters.
