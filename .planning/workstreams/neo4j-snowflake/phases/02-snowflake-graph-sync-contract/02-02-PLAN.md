<threat_model>
## Trust Boundaries

| Boundary | Description |
| --- | --- |
| Environment -> Snowflake connection | Runtime configuration creates authenticated Snowflake sessions. |
| CLI filters -> SQL generation | Operator-provided filter values affect graph materialization scope. |
| Executor -> Snowflake graph schema | Generated SQL mutates governed graph projection tables/views. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --- | --- | --- | --- | --- |
| T-02-02-01 | Spoofing | Snowflake connection settings | mitigate | Reuse existing `MDM_SNOWFLAKE_*` / `DBT_SNOWFLAKE_*` connection model; do not add `NEO4J_*` credential paths. |
| T-02-02-02 | Tampering | Filtered SQL generation | mitigate | Validate entity type and relationship type filters against fixed MDM/registry names before interpolating; keep identifiers passed through `_ident`. |
| T-02-02-03 | Repudiation | Graph sync result payload | mitigate | Return target schema, applied filters, and node/edge counts for command logs and SUMMARY evidence. |
| T-02-02-04 | Denial of Service | Unbounded graph materialization | mitigate | Preserve `--limit`, `--limit-per-type`, relationship type filters, and add entity type filters for repair workflows. |
| T-02-02-SC | Tampering | Package installs | accept | No package installs are planned in this plan. |
</threat_model>
