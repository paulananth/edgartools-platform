<threat_model>
## Trust Boundaries

| Boundary | Description |
| --- | --- |
| Operator CLI args -> graph sync executor | User-provided filters and target overrides influence Snowflake writes. |
| Environment -> CLI handler | Existing Snowflake settings authorize graph schema materialization. |
| Relationship derivation -> graph sync | `load-relationships` may derive MDM relationships before graph materialization. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| --- | --- | --- | --- | --- |
| T-02-03-01 | Spoofing | `sync-graph` runtime credentials | mitigate | Use Snowflake connection context only; tests assert no `NEO4J_*` credential dependency for `sync-graph`. |
| T-02-03-02 | Tampering | CLI target overrides | mitigate | Pass database/schema names through executor identifier validation; keep overrides narrow and explicit. |
| T-02-03-03 | Information Disclosure | Command logs and JSON output | mitigate | Output counts, table/schema names, and filters only; do not print passwords, tokens, `NEO4J_SECRET_JSON`, or Snowflake secrets. |
| T-02-03-04 | Denial of Service | Unbounded `sync-graph` execution | mitigate | Preserve `--limit`, `--limit-per-type`, `--relationship-type`, and add `--entity-type` for bounded repair and validation workflows. |
| T-02-03-SC | Tampering | Package installs | accept | No package installs are planned in this plan. |
</threat_model>
