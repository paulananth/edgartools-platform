# Milestone Audit: v1.1 Neo4j Bronze-To-Graph Pipe

## Verdict

**Status: gaps_found**

The currently available local `silver.duckdb` candidates do **not** contain all data required
to build and test the Neo4j company-person graph path.

They contain:

- Company entities via `sec_company`
- Company filing metadata via `sec_company_filing`
- Hundreds of thousands to over one million Forms 3/4/5 filing metadata rows

They do not contain:

- Parsed reporting people in `sec_ownership_reporting_owner`
- Ownership transaction facts in `sec_ownership_non_derivative_txn`
- Any joinable reporter-to-issuer rows for `IS_INSIDER` relationship derivation

## Why This Blocks Neo4j

The MDM person loader and `IS_INSIDER` derivation depend on these silver tables:

- `sec_company` for issuer/company nodes
- `sec_company_filing` for accession-to-issuer mapping
- `sec_ownership_reporting_owner` for reporting-person nodes
- `sec_ownership_non_derivative_txn` for ownership/security facts

The current silver data can create company nodes, but it cannot create people or
company-person relationships because `sec_ownership_reporting_owner` has zero rows.

## Required Closure

Phase 5 must first prove the source contract:

1. Confirm whether ownership XML artifacts exist in bronze for the Forms 3/4/5 filing metadata.
2. Run or repair the ownership bronze-to-silver parser path so `sec_ownership_reporting_owner`
   is populated.
3. Add a preflight check for `mdm run` / `mdm load-relationships` that fails when required
   source tables are empty.
4. Use a small fixture with nonzero `sec_company`, `sec_company_filing`, and
   `sec_ownership_reporting_owner` rows for repeatable MDM/Neo4j tests.
