<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Environment -> CLI | `MDM_SILVER_DUCKDB` and optional local cache path enter from environment variables. |
| CLI -> remote object storage | URI-backed DuckDB files are downloaded through `object_storage.read_bytes()`. |
| CLI -> MDM SQL database | `_session()` opens a mutable database boundary and must happen only after preflight succeeds. |
| CLI -> DuckDB SQL | Table validation queries DuckDB metadata/counts. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-13 | Information Disclosure | `MDM_SILVER_DUCKDB` URI reads | mitigate | Reuse `_silver_reader()` and `object_storage.read_bytes()` so unsupported protocols are rejected by `ALLOWED_REMOTE_PROTOCOLS`. |
| T-05-14 | Tampering | DuckDB table validation | mitigate | Use fixed required-table allowlists and table-name constants; do not accept table names from CLI args or environment. |
| T-05-15 | Tampering | MDM session/mutation | mitigate | Call preflight before `_session()` in all mutating silver-backed handlers. |
| T-05-16 | Information Disclosure | CLI logs | mitigate | Preserve `_safe_arguments()` and avoid printing `MDM_DATABASE_URL`, `NEO4J_PASSWORD`, or secret env values. |
| T-05-17 | Denial of Service | Large/remote silver reads | accept | URI-backed DuckDB download behavior already exists; this phase adds validation, not new remote protocols or unbounded scans beyond required table counts. |
| T-05-SC | Tampering | Package installs | accept | No new dependencies or package-manager install tasks are planned. |
</threat_model>
