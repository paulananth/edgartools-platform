<reachability_note>
## Edge-coverage reachability (READ — affects Wave 5 acceptance and the PIPE-05 wording flag)

The 11 `GRAPH_EDGE_*` views in `edgar_warehouse/mdm/snowflake_graph.py` all read from
`MDM_GRAPH_EDGES` filtered by `RELATIONSHIP_TYPE`. Each relationship type is derived from a
specific silver source (confirmed in `pipeline.py`):

| RELATIONSHIP_TYPE / GRAPH_EDGE table | Silver source | Reachable from Phase 5 ownership+ADV path? |
|--------------------------------------|---------------|--------------------------------------------|
| IS_INSIDER | ownership reporting owners | YES (guaranteed for any Form 3/4/5 issuer CIK) |
| HOLDS | ownership non-derivative txn | YES (guaranteed for a Form 4 issuer CIK) |
| ISSUED_BY | ownership securities | YES (guaranteed for a Form 4 issuer CIK) |
| COMPANY_HOLDS | ownership (corporate owner) | CONDITIONAL — needs a corporate reporting owner |
| IS_PERSON_OF | person/company resolution | CONDITIONAL — needs resolved person+company |
| MANAGES_FUND | ADV adviser -> fund | CONDITIONAL — needs the CIK to be an ADV filer with private funds |
| IS_ENTITY_OF | ADV adviser -> company | CONDITIONAL — needs the CIK to be an ADV filer |
| EMPLOYED_BY | DEF 14A proxy (`sec_executive_record`) | ONLY if fundamentals silver already populated (D-09/D-10: no new fetch) |
| AUDITED_BY | 10-K XBRL audit firm (`sec_accounting_flag` + audit firms) | ONLY if fundamentals silver + `mdm seed-audit-firms` present |
| INSTITUTIONAL_HOLDS | 13F (`sec_thirteenf_holding`) | ONLY if fundamentals silver already populated |
| HAS_PARENT_COMPANY | `mdm_company.parent_company_entity_id` | UNREACHABLE by construction — no ownership/ADV/fundamentals loader sets this column |

Consequence: a hard "all 11 GRAPH_EDGE_* >= 1 row" gate is NOT provable in Phase 5 via the
existing-silver ownership+ADV path. `HAS_PARENT_COMPANY` is unreachable by construction;
EMPLOYED_BY/AUDITED_BY/INSTITUTIONAL_HOLDS are reachable only if fundamentals silver is already
present in the selected silver source. D-29 explicitly assigns "all 11 edges for all active CIKs"
to Phase 6.

Wave 5 therefore uses a **parity gate** (what Phase 5 actually proves — that the MDM -> Snowflake
sync path is faithful), not a raw count of all 11 edges:
  PASS = (a) `MDM_MINUS_GRAPH = 0` for every active relationship type (sync materialized exactly
            what MDM derived — see `render_validation` `active_mdm_relationship_parity`), AND
         (b) the ownership-core edges `IS_INSIDER`, `HOLDS`, `ISSUED_BY` are nonzero (proves real
            data traversed the full pipe; guaranteed for any Form 4 issuer CIK).
Any zero-count edge gets a documented reason in VALIDATION.md (mirrors the D-24/D-28 documented-
exclusion pattern). This satisfies "do not silently weaken a requirement": the residual PIPE-05
"all 11" clause is flagged to the orchestrator for re-wording or acceptance of the parity gate.
</reachability_note>
