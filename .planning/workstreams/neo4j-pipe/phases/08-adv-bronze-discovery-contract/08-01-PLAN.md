<objective>
Create the Phase 8 ADV bronze discovery/read contract.

This plan adds a small helper module and focused tests proving existing ADV bronze artifacts can
be selected from registry rows or explicit operator-provided artifact records, read through the
storage adapter, and reported with structured missing/unreadable issues. It does not add the
`parse-adv-bronze` CLI command and does not parse or merge ADV rows into silver; those belong
to Phase 9.
</objective>

<threat_model>
  <threat id="T-08-01" severity="medium" surface="explicit artifact input">
    Operator-provided paths could try unsupported protocols or path traversal. Mitigation: do not
    implement custom file access; route reads through `object_storage.read_bytes`, whose protocol
    allowlist handles remote schemes. Tests inject `read_bytes_fn` and do not bypass the adapter.
  </threat>
  <threat id="T-08-02" severity="medium" surface="form/table selection">
    Dynamic form or table identifiers could lead to unsafe SQL or wrong parser-family selection.
    Mitigation: use a fixed ADV form allowlist and fixed SQL table names only.
  </threat>
  <threat id="T-08-03" severity="high" surface="network side effects">
    Backfill discovery could accidentally re-fetch missing SEC filings. Mitigation: the helper must
    not import/call `download_sec_bytes`, `_download_sec_bytes`, or `refresh_filing_artifacts`;
    tests patch these helpers to fail if invoked.
  </threat>
</threat_model>

  <action>
    Create `edgar_warehouse/application/adv_bronze_discovery.py` with fixed ADV form allowlist
    `{"ADV","ADV/A","ADV-E","ADV-E/A","ADV-H","ADV-H/A","ADV-NR","ADV-W","ADV-W/A"}` and pure
    helper functions. Provide immutable data structures for discovered candidates, read payloads,
    and issues. Implement `discover_adv_bronze_artifacts(db, accession_list=None, explicit_artifacts=None, limit=None)`
    so it:
    - queries fixed table `sec_company_filing` for `accession_number`, `cik`, and `form` where `form` is in the ADV allowlist;
    - filters by `accession_list` when supplied;
    - applies `limit` after filtering;
    - uses `db.get_filing_attachments(accession_number)` and `db.get_raw_object(raw_object_id)` for registry candidates;
    - records issues for missing primary attachment, missing raw object, and empty storage path;
    - accepts explicit artifact records with `accession_number`, `form`, optional `cik`, and `storage_path`;
    - rejects or reports explicit artifact records whose `form` is not in the ADV allowlist;
    - never lists S3 prefixes and never calls SEC download helpers.
    Implement `read_adv_bronze_artifacts(candidates, read_bytes_fn=read_bytes)` so it reads each
    candidate's `storage_path`, returns payloads for successes, and records unreadable-path issues
    while continuing through the batch.
  </action>
