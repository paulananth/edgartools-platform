## Context

MANAGES_FUND, IS_ENTITY_OF, IS_PERSON_OF). Three more types exist in `pipeline.py` but have
NO positive-path tests:

- **EMPLOYED_BY** â€” reads `sec_executive_record` (DEF 14A proxy)

test would crash with `KeyError("Unknown relationship type")` before reaching the deriver.

the remaining 3 are silently excluded from idempotency validation.

**No pipeline.py changes are needed.** All three derivers are fully implemented and correct.
This plan is 100% test-only.

```python
def test_audited_by_change_detection(self, session, fixture_world):
    """AUDITED_BY: auditor_changed=True when fiscal_year N+1 has a different auditor. (REL-?)"""
    # Seed a second audit firm for the change-detection test
    pwc_entity_id = _add_entity(session, "audit_firm")
    session.add(MdmAuditFirm(
        entity_id=pwc_entity_id,
        firm_name="PricewaterhouseCoopers LLP",
        canonical_name="PricewaterhouseCoopers LLP",
        pcaob_firm_id="E2",
        big4=True,
    ))
    session.commit()

    silver = StubSilver({
        "sec_accounting_flag": [
            # FY2022: first for CIK 910001 â€” auditor_changed must be False
            {"cik": 910001, "accession_number": "acc-2022", "fiscal_year": 2022,
             "period_end": None, "auditor_pcaob_id": "E1",
             "auditor_name": "Deloitte LLP", "icfr_attestation": True},
            # FY2023: same CIK, different auditor â€” auditor_changed must be True
            {"cik": 910001, "accession_number": "acc-2023", "fiscal_year": 2023,
             "period_end": None, "auditor_pcaob_id": "E2",
             "auditor_name": "PricewaterhouseCoopers LLP", "icfr_attestation": True},
        ],
    })
    pipe = MDMPipeline(session=session, silver=silver)
    summary = pipe.derive_relationships(relationship_types=["AUDITED_BY"])
    assert summary["AUDITED_BY"]["inserted"] == 2

    # Verify auditor_changed property on the two relationship instances
    from sqlalchemy import select
    instances = session.scalars(
        select(MdmRelationshipInstance).order_by(MdmRelationshipInstance.effective_from)
    ).all()
    # FY2022 row: no prior year â†’ auditor_changed = False
    fy2022 = next(i for i in instances if i.properties.get("fiscal_year") == 2022)
    fy2023 = next(i for i in instances if i.properties.get("fiscal_year") == 2023)
    assert fy2022.properties["auditor_changed"] is False
    assert fy2023.properties["auditor_changed"] is True
```

**GREEN criterion:** AUDITED_BY deriver processes rows in ORDER BY cik, fiscal_year; FY2022 has no prev_cik match â†’ `auditor_changed=False`; FY2023 has same prev_cik, different `prev_auditor_name` â†’ `auditor_changed=True`. âś“ (No pipeline changes needed â€” logic exists at lines 1207-1213.)
