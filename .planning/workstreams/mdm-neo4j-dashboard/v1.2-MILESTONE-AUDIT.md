## Cross-Phase Integration

Affected requirements: `DASH-02`, `DASH-03`, `ISO-02`, `MDM-01`, `MDM-02`, `GRAPH-02`, `GRAPH-03`.

### Blocker: Neo4j Node Count Display Mapping

`examples/mdm_graph_dashboard/streamlit_app.py` renders entity-domain coverage by deriving a Neo4j node label with:

```python
graph_key = label.rstrip("s")
```

This turns `Companies` into `Companie`, `Securities` into `Securitie`, and leaves `People` unchanged. Valid graph metric keys are registry labels such as `Company`, `Security`, and `Person`, so Neo4j node counts can display as zero even when the helper returned correct counts.

Affected requirements: `GRAPH-01`, `UX-01`.

Suggested closure:

- Insert or execute a closure phase for the display mapping bug.
- Use registry `neo4j_label` values or entity type details to map each MDM domain to the correct Neo4j key.
- Add a regression test that `Companies -> Company`, `Securities -> Security`, and `People -> Person` render using the returned node counts.
