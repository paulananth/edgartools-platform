<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Streamlit shell -> Neo4j helper | Dashboard UI calls optional graph helper code using environment configuration. |
| Environment -> Neo4j client | Optional `NEO4J_*` and `NEO4J_SECRET_JSON` values are sensitive and must not leak into UI/status output. |
| Helper -> Neo4j | Cypher crosses into graph storage and must remain read-only. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-04 | Tampering | `edgar_warehouse/mdm/graph_readonly.py` | mitigate | Use only static `RETURN 1 AS ok` smoke query in Phase 8 and test captured Cypher for absence of write tokens. |
| T-08-05 | Information Disclosure | Neo4j config/error handling | mitigate | Return exact UI-SPEC copy and status labels; never render raw URI, username, password, database, secret JSON, hostnames, or raw exceptions. |
| T-08-06 | Elevation of Privilege | Graph helper imports | mitigate | Static tests block `GraphSyncEngine`, merge Cypher helpers, sync/backfill APIs, and any write-oriented graph API imports. |
| T-08-SC | Tampering | npm/pip/cargo installs | accept | No package-manager install task is planned; existing `uv` extras are used. |
</threat_model>
