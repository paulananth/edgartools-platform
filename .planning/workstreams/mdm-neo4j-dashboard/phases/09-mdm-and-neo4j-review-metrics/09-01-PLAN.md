<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Streamlit caller -> MDM helper | Dashboard code requests metrics and receives sanitized structured data. |
| MDM helper -> SQL database | Helper crosses into relational data; must perform SELECT-only bounded reads. |
| SQL/driver failure -> operator payload | Exceptions can contain sensitive connection data and must be replaced with fixed copy. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | `get_mdm_dashboard_metrics` | mitigate | Enforce read-only behavior with SELECT-only SQLAlchemy queries, injected-session no-commit tests, owned-session rollback/close, and static blocked-token tests. |
| T-09-02 | Information Disclosure | MDM failure payloads | mitigate | Return fixed `MDM_UNAVAILABLE_MESSAGE`, env var names only, and tests excluding DSNs, users, passwords, hosts, and raw exception text. |
| T-09-03 | Denial of Service | Pending sync diagnostics | mitigate | Clamp `per_type_sample_limit` and `global_sample_limit` before constructing queries; tests prove bounded output. |
| T-09-04 | Tampering | Dynamic Cypher label/type validation | accept | This plan does not compose Cypher; Plan 09-02 owns registry-backed Cypher validation. |
| T-09-05 | Elevation of Privilege | Mutation controls | mitigate | Helper module exposes no mutation functions and imports no pipeline, sync, migration, resolver, or stewardship surfaces. |
| T-09-SC | Tampering | Package installs | accept | No package-manager install tasks exist; package legitimacy audit says no new packages are planned. |
</threat_model>
