<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Streamlit/MDM registry -> graph helper | Registry labels and relationship types become Cypher structure after validation. |
| Graph helper -> Neo4j | Helper crosses into graph data; only read queries are allowed. |
| Neo4j/driver failure -> operator payload | Driver failures can include secrets and must be normalized. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-06 | Tampering | Dynamic Cypher labels/types | mitigate | Validate all labels and relationship types before interpolation; reject unsafe identifiers before `session.run`. |
| T-09-07 | Tampering | Neo4j metric queries | mitigate | Generate only `MATCH` and `RETURN` read queries; static tests block write Cypher tokens and graph sync imports. |
| T-09-08 | Information Disclosure | Neo4j unavailable/query-failed results | mitigate | Reuse fixed safe copy and env var names only; tests exclude URI, user, password, host, secret JSON, and raw exception text. |
| T-09-09 | Denial of Service | Missing/extra graph diagnostics | mitigate | Enforce per-type and global caps in helper functions before looping or issuing Cypher. |
| T-09-10 | Elevation of Privilege | No mutation controls | mitigate | Do not import sync, merge, backfill, or CLI mutation surfaces; no helper returns mutation action descriptors. |
| T-09-SC | Tampering | Package installs | accept | No package-manager install tasks exist; package legitimacy audit says no new packages are planned. |
</threat_model>
