<decision_coverage>
This plan renders and verifies the locked Phase 9 UI/diagnostic decisions:
- D-01: Overview leads with MDM entity counts, MDM relationship counts, Neo4j node/edge counts, and pending sync totals.
- D-02: Snapshot metrics include count plus simple status.
- D-03: Attention-needed signals cover sync gaps, coverage gaps, unavailable sources, missing edges, and extra graph data; zero counts alone are not warnings.
- D-04: Detailed metrics are organized into MDM Overview, Neo4j Overview, and Graph Coverage.
- D-05: Overview owns the coverage snapshot; detailed metrics live in existing sidebar destinations.
- D-06: Raw counts and meaningful percentages are shown, especially relationship coverage percent.
- D-07: Neo4j unavailable states do not block MDM metrics.
- D-08: Lightweight Streamlit-native charts are allowed without delaying metric tables.
- D-09: Phase 8 smoke output is removed or demoted after real metrics exist.
- D-10: Refresh behavior is section-aware where Streamlit can do so cleanly.
- D-11: The implementation may choose the cleanest refresh control as long as section refresh is not awkward.
- D-12: Last-refreshed timestamps appear per section.
- D-13: Metric failures and attention signals are gathered into one warning area.
- D-14: Warning area groups blocking failures before non-blocking coverage warnings.
- D-15: Graph Coverage starts with entity-domain coverage before relationship edge types.
- D-16: Entity-domain coverage is chart-first with a details table underneath.
- D-17: Relationship coverage table includes MDM active count, Neo4j edge count, pending sync count, missing estimate, and coverage percent.
- D-18: Missing estimate is `max(MDM active count - Neo4j edge count, 0)`.
- D-19: All active registered relationship types render, including zero-row types.
- D-20: Extra graph data warnings appear when Neo4j edge counts exceed MDM active rows.
- D-21: MDM readiness and graph sync readiness render as separate warning groups.
- D-22: MDM readiness severity distinguishes structural problems from sparse data.
- D-23: Graph sync readiness warns for pending sync, Neo4j unavailable, lower Neo4j counts, and extra graph data.
- D-24: Warning severity uses error, warning, and info.
- D-25: Warnings include short operator action text but no mutation buttons.
- D-26: Counts include bounded sample rows for pending sync and missing/extra graph diagnostics.
- D-27: Sample rows include cheap operator-readable names, falling back to IDs.
- D-28: Samples use small per-type limits capped globally.
- D-29: Samples follow registry order and oldest-row ordering within type.
- D-30: Samples do not expose raw properties JSON by default.
</decision_coverage>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Helper payloads -> Streamlit UI | Structured metrics are rendered to operators without direct query construction. |
| Operator refresh -> cached metric reads | Refresh clears cached data and reruns read-only helpers, not mutation controls. |
| Dashboard text/docs -> operator actions | Copy can encourage safe external actions but must not offer dashboard mutation buttons or new deployment paths. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-11 | Tampering | Streamlit data access | mitigate | Streamlit imports helper modules only; architecture tests forbid raw SQL, raw Cypher, sync handlers, and mutation surfaces. |
| T-09-12 | Information Disclosure | Dashboard errors and docs | mitigate | Render helper safe copy only; README and UI name env vars without values or raw connection strings. |
| T-09-13 | Tampering | Dynamic Cypher label/type validation | mitigate | UI passes registry-backed helper inputs and never composes labels/types directly; Plan 09-02 helper validators remain the enforcement point. |
| T-09-14 | Denial of Service | Diagnostic rendering | mitigate | UI renders capped helper sample payloads and bounded-sample copy; no unbounded filters or free-text query controls are introduced. |
| T-09-15 | Elevation of Privilege | No mutation controls | mitigate | Architecture tests block sync, derive, load, migrate, seed, merge, repair, quarantine, accept, reject, edit, delete, credential, deployment, Terraform, Step Functions, Snowflake/dbt, and rollout controls. |
| T-09-SC | Tampering | Package installs | accept | No package-manager install tasks exist; package legitimacy audit says no new packages are planned. |
</threat_model>
