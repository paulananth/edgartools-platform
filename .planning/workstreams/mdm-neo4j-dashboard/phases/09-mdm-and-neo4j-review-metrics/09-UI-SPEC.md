## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline label-value gaps, status label spacing |
| sm | 8px | Compact spacing between metrics, captions, and table legends |
| md | 16px | Default vertical rhythm between controls, warning groups, and tables |
| lg | 24px | Section grouping gaps after headers and before related metric groups |
| xl | 32px | Major split between snapshot, warning area, and detail tables |
| 2xl | 48px | Page-level separation when changing from overview metrics to diagnostics |
| 3xl | 64px | Reserved; avoid in dense operator views unless a page section fully resets context |

Exceptions: Streamlit's built-in sidebar width, dataframe row height, metric tile dimensions, chart height, and button dimensions may use framework defaults. Sidebar navigation and refresh controls must keep at least 44px click height where Streamlit exposes sizing.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FFFFFF | Main dashboard background and primary content area |
| Secondary (30%) | #F6F8FA | Sidebar, dataframe header bands, subtle section dividers, empty-state surfaces |
| Accent (10%) | #1F77B4 | Selected navigation state, refresh focus/active state, domain comparison bars, status links, and connected Neo4j status |
| Destructive | #D92D20 | Error states only; no destructive actions exist in this phase |

Accent reserved for: selected sidebar item, refresh button focus/active state, inline diagnostic references, entity-domain comparison bars, and connected Neo4j status. Do not use accent for every metric tile, every table row, or decorative backgrounds.

Status colors:

| Status | Value | Usage |
|--------|-------|-------|
| Success | #12B76A | OK metrics, connected sources, complete coverage rows |
| Warning | #F79009 | Pending sync, missing graph data, extra graph data, sparse readiness signals |
| Error | #D92D20 | Blocking MDM failure, unavailable required source, query failure, invalid configuration |
| Info | #667085 | Zero counts that are not failures, unavailable optional detail, bounded sample explanations |

All status indicators must include visible text such as `OK`, `Pending sync`, `Missing graph data`, `Extra graph data`, or `Unavailable`; color must never be the only signal.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Refresh metrics |
| Empty state heading | No MDM metrics available |
| Empty state body | Load or seed the MDM database, then refresh metrics. |
| Error state | MDM metrics are unavailable. Check `MDM_DATABASE_URL`, confirm the database is reachable, and refresh metrics. |
| Destructive confirmation | none: this phase has no destructive actions and must not render mutation controls |

Additional required copy:

| Scenario | Copy |
|----------|------|
| Neo4j unavailable | Neo4j metrics are unavailable. MDM metrics remain available. Check Neo4j configuration and network access. |
| Pending sync warning | Pending graph sync rows exist. Run the existing graph sync process outside this dashboard when ready. |
| Missing graph data warning | MDM active relationships exceed Neo4j edges for this type. Review pending sync and graph sync status. |
| Extra graph data warning | Neo4j edges exceed active MDM relationships for this type. Review graph freshness and sync history. |
| Zero count info | No rows found for this category. Zero counts alone are not warnings. |
| Bounded samples note | Showing a bounded sample for review. Counts may include additional rows. |
| No mutation reminder | This dashboard is read-only and does not run sync, repair, migrate, or load commands. |

Tone: operator-facing, factual, short. Avoid marketing language. Do not imply the dashboard can repair, sync, migrate, edit, accept, reject, or mutate data.

## Layout Contract

Phase 09 must preserve the Phase 08 wide-layout Streamlit shell:

| Area | Contract |
|------|----------|
| Page config | `page_title="EdgarTools MDM Graph"`, `layout="wide"` |
| Sidebar title | EdgarTools MDM |
| Sidebar caption | Read-only MDM and Neo4j status |
| Sidebar navigation | Overview, Entities, Relationships, Graph Coverage, Neighborhood |
| Sidebar action | `Refresh metrics` clears metric caches and reruns the active section |
| Source diagnostics | Show MDM and Neo4j status labels without credentials, full connection strings, or secret values |

View ownership:

| Sidebar destination | Required Phase 09 surface |
|---------------------|---------------------------|
| Overview | Coverage snapshot first, then grouped warning area, then section last-refreshed timestamps. Demote Phase 08 smoke output to a collapsed or low-priority diagnostic surface, or remove it from the primary view. |
| Entities | MDM Overview detail: entity counts by domain for company, adviser, person, security, and fund. Include raw counts and source-readiness status. |
| Relationships | MDM Overview detail: relationship counts by registered relationship type, active count, pending sync count, and readiness status. Show every registered active relationship type, including zero-count types. |
| Graph Coverage | Graph Coverage detail: chart-first entity-domain comparison, relationship coverage table, pending sync samples, missing-edge samples, and extra graph data samples. |
| Neighborhood | Keep the Phase 08 placeholder. Entity neighborhood lookup and graph visualization remain out of scope for Phase 09. |

Overview snapshot:

| Snapshot metric | Display contract |
|-----------------|------------------|
| MDM entities | Count plus simple status: `OK`, `No rows`, or `Unavailable` |
| MDM relationships | Count plus simple status: `OK`, `Pending sync`, `No rows`, or `Unavailable` |
| Neo4j nodes | Count plus simple status: `OK`, `Missing graph data`, or `Unavailable` |
| Neo4j edges | Count plus simple status: `OK`, `Missing graph data`, `Extra graph data`, or `Unavailable` |
| Pending sync | Count plus status: `OK` when zero, `Pending sync` when greater than zero |

Use `st.metric` in a two- or three-column layout for the snapshot. Do not hide MDM metrics when Neo4j is unavailable. If MDM is unavailable, show the blocking error state and skip metric tables that would use stale or missing MDM data.

Grouped warning area:

| Group | Order | Contract |
|-------|-------|----------|
| Blocking failures | First | MDM unavailable, invalid required configuration, or required query failure. Use `st.error`. |
| Non-blocking coverage warnings | Second | Neo4j unavailable, pending sync, missing graph data, extra graph data, sparse readiness signals. Use `st.warning` for warnings and `st.info` for informational zero-count/sparse states. |

Each warning row must include severity, short problem text, affected source or relationship type, and short operator action text. Do not repeat the same warning inline in every table when it is already present in the grouped warning area; table rows may show compact status labels.

## Interaction Contract

Refresh behavior:

| Control | Contract |
|---------|----------|
| Refresh metrics | Primary CTA. Clears cached Phase 09 metric result sets and reruns the active section. |
| Section refresh | Prefer separate MDM, Neo4j, and Graph Coverage refresh controls when Streamlit implementation stays clean. If section buttons are awkward, keep the global refresh and still show per-section last-refreshed timestamps. |
| Last refreshed | Show an ISO-like local timestamp per MDM, Neo4j, and Graph Coverage section when that section has successfully loaded. Use `Unavailable` instead of fake timestamps. |

Data access:

| Contract | Requirement |
|----------|-------------|
| Read-only only | Metric helpers must use read-only MDM and Neo4j access. The UI must not call graph sync, merge, derive, migrate, seed, load, repair, accept/reject, or pipeline handlers. |
| MDM required | MDM metrics are the foundation. If MDM is unavailable, render a blocking error and no stale metric tables. |
| Neo4j optional | If Neo4j is unavailable, keep MDM Overview and Relationships usable. Mark Neo4j Overview and Graph Coverage as unavailable or partial. |
| Bounded queries | Pending sync, missing-edge, and extra graph samples must be bounded by per-type limits and a global cap. |
| Registry order | Relationship type tables and sample groups must follow relationship registry order. |
| Safe dynamic Cypher | Relationship types and labels must be validated against the registry before any dynamic Cypher is composed. |

Forbidden UI:

| Forbidden control | Reason |
|-------------------|--------|
| Sync graph, derive relationships, load relationships, migrate, seed universe, merge, repair, quarantine, accept/reject review, or mutation buttons | Phase 09 is read-only |
| Secret editing, credential display, or full connection-string display | Security and operator-safety requirement |
| Generated deployment JSON, Terraform, Step Functions, Snowflake, dbt, or runtime rollout controls | Explicitly out of scope |
| Unbounded filters, free-text Cypher, SQL editors, or raw property JSON by default | Review safety and bounded-query requirement |

## Metrics And Data Display Contract

Entity domains:

| Domain key | Visible label | Required in comparison |
|------------|---------------|------------------------|
| company | Companies | yes |
| adviser | Advisers | yes |
| person | Persons | yes |
| security | Securities | yes |
| fund | Funds | yes |

MDM Overview:

| Surface | Columns or values |
|---------|-------------------|
| Entity counts table | Domain, MDM count, status, last refreshed |
| Relationship counts table | Relationship type, active count, pending sync count, total count if available, status |
| Source-readiness warnings | Severity, source, issue, operator action |

Neo4j Overview:

| Surface | Columns or values |
|---------|-------------------|
| Node counts table | Label, Neo4j count, status, last refreshed |
| Relationship counts table | Relationship type, Neo4j edge count, status, last refreshed |
| Unavailable state | Preserve MDM content and show Neo4j unavailable copy |

Graph Coverage:

| Surface | Contract |
|---------|----------|
| Entity-domain comparison chart | Chart-first MDM vs Neo4j bars by company, adviser, person, security, and fund. Use a Streamlit-native bar chart backed by the details table below. |
| Entity-domain details table | Domain, MDM count, Neo4j count, difference, coverage percent, status |
| Relationship coverage table | Relationship type, MDM active count, Neo4j edge count, pending sync count, missing estimate, coverage percent, status |
| Pending sync samples | Relationship type, source identifier, source name when cheap, target identifier, target name when cheap, created/updated age or ordering timestamp when available |
| Missing-edge samples | Relationship type, source identifier, source name when cheap, target identifier, target name when cheap, MDM relationship identifier when available |
| Extra graph data samples | Relationship type, source identifier, source name when cheap, target identifier, target name when cheap, Neo4j edge identifier when available |

Coverage rules:

| Rule | Contract |
|------|----------|
| Missing estimate | `max(MDM active count - Neo4j edge count, 0)` |
| Extra graph data | When `Neo4j edge count > MDM active count`, show warning status and include bounded samples when available |
| Coverage percent | When MDM active count is greater than zero, calculate `min(Neo4j edge count / MDM active count, 1.0) * 100`; when MDM active count is zero, show `-` and status `No MDM active rows` |
| Zero counts | Zero counts alone are informational, not warnings |
| Registered types | Show all registered active relationship types, including zero MDM active rows |

Numeric formatting:

| Value | Format |
|-------|--------|
| Counts | Thousands separators, no decimals |
| Percentages | One decimal place; use `-` when denominator is zero or source unavailable |
| Dates | ISO date `YYYY-MM-DD` |
| Timestamps | ISO-like local display with timezone when available |
| Missing values | `-` in tables |
| Sample limits | Show per-type small limits, default 5 rows per relationship type, with a global cap visible in helper text |

Raw properties JSON must not appear by default. Show key identifiers and names only unless a later phase adds an explicit detail surface.

## Accessibility And Responsiveness

| Area | Contract |
|------|----------|
| Layout | Must remain usable at 1280px desktop width and 390px mobile width using Streamlit native responsive stacking. |
| Tables | Use `st.dataframe(..., use_container_width=True, hide_index=True)`; wide tables must scroll horizontally inside the widget rather than forcing page overflow. |
| Charts | Chart data must be duplicated in an adjacent or immediately following table. Charts must not be the only source of a value. |
| Status | Use visible status text with color. Do not rely on red/yellow/green alone. |
| Controls | Every input/control must have a visible label. Do not use placeholder-only labels. |
| Keyboard | All controls must remain Streamlit-native and keyboard reachable. Do not add custom HTML controls. |
| Text fit | Table labels, metric labels, and warning text must wrap cleanly; do not use viewport-scaled font sizes or negative letter spacing. |
