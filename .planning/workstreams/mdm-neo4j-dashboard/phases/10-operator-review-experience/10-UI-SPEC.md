## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline label-value gaps and compact caption spacing |
| sm | 8px | Control groups, table captions, status label spacing |
| md | 16px | Default vertical rhythm between filters, tables, and warnings |
| lg | 24px | Section grouping gaps after headings and before major table groups |
| xl | 32px | Gaps between attention summary, snapshot metrics, and page detail sections |
| 2xl | 48px | Page-level separation only when moving to a different review task |
| 3xl | 64px | Reserved; avoid in dense operator review views unless context fully resets |

Exceptions: Streamlit's built-in sidebar width, dataframe row height, metric tile sizing, chart sizing, and form control dimensions may use framework defaults. Sidebar controls and refresh action must retain Streamlit-native accessible click targets, with 44px minimum height where the framework exposes sizing.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 | 1.5 |
| Label | 14px | 400 | 1.4 |
| Heading | 20px | 600 | 1.2 |
| Display | 28px | 600 | 1.2 |

Use these four sizes only for any custom Markdown/CSS. Prefer Streamlit defaults where they map closely. Navigation labels and table column headings use title case. Captions, empty states, warnings, and runbook prose use sentence case. Do not scale fonts with viewport width.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #FFFFFF | Main dashboard background and primary content area |
| Secondary (30%) | #F6F8FA | Sidebar, dataframe header bands, subtle section dividers, empty-state surfaces |
| Accent (10%) | #1F77B4 | Selected navigation state, `Refresh metrics` focus/active state, filter focus state, connected Neo4j status links if Streamlit renders links |
| Destructive | #D92D20 | Blocking error and permission-error states only; no destructive actions exist in this phase |

Accent reserved for: selected sidebar item, `Refresh metrics` focus/active state, active filter focus ring when exposed by Streamlit, and safe inline links in operator docs. Do not use accent for every metric, table row, warning, or decorative background.

Status colors:

| Status | Value | Streamlit Primitive | Usage |
|--------|-------|---------------------|-------|
| Success | #12B76A | `st.success` or positive metric delta text | `OK`, connected sources, no attention-needed items |
| Warning | #F79009 | `st.warning` | Pending sync, missing graph data, extra graph data, Neo4j unavailable, partial review state |
| Info | #667085 | `st.info` | Empty filtered rows, neutral sparse data, bounded sample explanations |
| Error | #D92D20 | `st.error` | Missing MDM configuration, MDM connection failure, permission-denied failures |

All status indicators must include visible text. Color must never be the only signal.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Refresh metrics |
| Empty state heading | No rows match the current filters. |
| Empty state body | Adjust the selected type or row limit, then review the table again. |
| Error state | MDM database unavailable. Check `MDM_DATABASE_URL`, confirm the database is reachable, and restart the dashboard. |
| Destructive confirmation | none: this phase has no destructive actions and must not render mutation controls |

Required state copy:

| Scenario | Copy |
|----------|------|
| Missing MDM configuration | MDM configuration is required. Set `MDM_DATABASE_URL`, then restart the dashboard. |
| MDM permission error | MDM database permission denied. Confirm the configured database user can run read-only SELECT queries. |
| Neo4j unavailable | Neo4j graph metrics unavailable. MDM overview remains available. |
| Neo4j permission error | Neo4j permission denied. Confirm the configured graph user can run read-only MATCH queries. |
| Filtered empty table | No rows match the current filters. |
| Unfiltered empty MDM data | No MDM rows were returned for this review surface. Confirm the MDM database has been populated before reviewing graph coverage. |
| Pending sync warning | Pending sync rows exist. Review the bounded samples and run existing operator checks outside the dashboard. |
| Missing graph data warning | Missing Neo4j edges were estimated for this relationship type. Review pending sync and missing-edge samples. |
| Extra graph data warning | Neo4j has more edges than active MDM rows for this relationship type. Review extra graph samples against the MDM registry. |
| Bounded sample caption | Samples are bounded diagnostics, not exhaustive diffs. |
| Read-only guarantee | This dashboard does not run sync, repair, migrate, load, or write actions. |

Secret-safe error details may name expected environment variables and a next action only. They must not render hostnames, usernames, passwords, secret JSON payloads, raw database URLs, full Neo4j URIs, or raw driver exception text.

## Navigation Contract

The dashboard must be an operational review tool, not a landing page. Use `st.set_page_config(page_title="EdgarTools MDM Graph", layout="wide")`.

| Area | Contract |
|------|----------|
| Sidebar title | EdgarTools MDM |
| Sidebar caption | Read-only MDM and Neo4j status |
| Primary navigation | `Overview`, `MDM Overview`, `Neo4j Overview`, `Mismatch Diagnostics` in this exact order |
| Removed or renamed routes | Existing `Entities`, `Relationships`, `Graph Coverage`, and `Neighborhood` must not remain as primary navigation labels |
| Global row limit | Sidebar `selectbox` labeled `Row limit`; choices `25`, `50`, `100`, `250`; default `50` |
| Global action | Sidebar button labeled `Refresh metrics`; clears cached read-only metric data and reruns |
| Source status | Sidebar or top-of-page status may show MDM required and Neo4j optional state, but must not reveal credentials or connection strings |

Landing behavior: Operators must land on `Overview`. The first content should be attention-needed items, then high-level counts.

## Layout Contract

| View | Required layout |
|------|-----------------|
| Overview | Title `EdgarTools MDM Graph`; caption `Read-only MDM and Neo4j coverage review.`; attention-needed summary first; then snapshot metrics for MDM entities, MDM relationships, Neo4j nodes, Neo4j edges, pending sync, missing graph data, and extra graph data where available |
| MDM Overview | Page-specific filters first; MDM entity and relationship summary tables second; MDM readiness warnings and last-refreshed caption nearby; no Neo4j-only tables |
| Neo4j Overview | Neo4j availability state first; page-specific filters; node counts by registry label and relationship counts by type; if Neo4j is unavailable, render only unavailable state and keep MDM navigation usable |
| Mismatch Diagnostics | Entity-domain coverage first; relationship coverage table second; bounded pending sync, missing-edge, and extra graph sample tables third |

Density: Prefer compact, table-first review surfaces. Do not use nested cards, marketing hero sections, large decorative empty regions, custom HTML dashboards, graph visualizations, or oversized typography. Use `st.divider()` only when the next area changes task context.

Attention summary contract:

| Group | Order | Contents |
|-------|-------|----------|
| Blocking failures | 1 | Missing MDM config, MDM connection failure, MDM permission failure |
| Coverage warnings | 2 | Neo4j unavailable, pending sync, missing graph data, extra graph data |
| Informational notices | 3 | Sparse-but-valid data, empty filtered rows, bounded sample explanation |

If no blocking failures or warnings exist, render `No blocking failures or coverage warnings.`

## Filter Contract

Use a hybrid filter model: one global row limit in the sidebar, plus page-specific single-select controls in the relevant view.

| Control | Location | Values | Default | Applies to |
|---------|----------|--------|---------|------------|
| Row limit | Sidebar | `25`, `50`, `100`, `250` | `50` | Diagnostic and sample tables; any large detail table |
| Entity type | MDM Overview | `All` plus registry entity types/domains | `All` | MDM entity table/detail rows |
| Entity type | Neo4j Overview | `All` plus registry entity types/domains mapped to Neo4j labels | `All` | Neo4j node count table |
| Entity type | Mismatch Diagnostics | `All` plus registry entity types/domains | `All` | Entity-domain coverage rows |
| Relationship type | MDM Overview | `All` plus registered relationship types | `All` | MDM relationship count table |
| Relationship type | Neo4j Overview | `All` plus registered relationship types | `All` | Neo4j relationship count table |
| Relationship type | Mismatch Diagnostics | `All` plus registered relationship types | `All` | Relationship coverage and bounded sample tables |

Filter rules:

- Use `st.selectbox` only; do not use text inputs, multiselects, checkboxes, toggles, or mutation-like action controls.
- Options must come from structured MDM registry/metrics payloads, not free-form operator input.
- `All` must preserve registry ordering where available.
- Filtered empty results must show `No rows match the current filters.`
- Aggregate snapshot metrics on `Overview` must remain unfiltered so operators can see system-wide attention first.

## Table Behavior

Use `st.dataframe(..., use_container_width=True, hide_index=True)` for all review tables.

| Table | Required columns |
|-------|------------------|
| MDM entity counts | `Domain`, `Count`, `Status` |
| MDM relationship counts | `Relationship Type`, `MDM Active`, `Pending Sync`, `Status` |
| Neo4j node counts | `Entity Type`, `Neo4j Label`, `Neo4j Nodes`, `Status` |
| Neo4j relationship counts | `Relationship Type`, `Neo4j Edges`, `Status` |
| Entity-domain coverage | `Domain`, `MDM Count`, `Neo4j Label`, `Neo4j Count`, `Status` |
| Relationship coverage | `Relationship Type`, `MDM Active`, `Neo4j Edges`, `Pending Sync`, `Missing Estimate`, `Coverage`, `Status` |
| Pending sync samples | `Relationship Type`, `Source`, `Target`, `Created` |
| Missing-edge samples | `Relationship Type`, `Source`, `Target`, `Created` |
| Extra graph samples | `Relationship Type`, `Source`, `Target`, `Created` |

Formatting:

| Value | Format |
|-------|--------|
| Counts | Thousands separators, no decimals |
| Percentages | One decimal place, `-` when unavailable |
| Dates | ISO date `YYYY-MM-DD` |
| Timestamps | ISO-like timestamp with timezone when available |
| Missing values | `-` |

Sorting and limits:

- Relationship types should use registry order where available, then stable alphabetical fallback.
- Samples should preserve existing helper priority: registry order, then oldest rows within type.
- Row limit applies after filters and before rendering large diagnostic/sample tables.
- Aggregate registry count tables should show all registered types unless the operator applies a page filter.
- Wide tables must scroll inside the dataframe widget rather than forcing page overflow.

## GRAPH-01 Display Correctness

Phase 10 must close the carried `GRAPH-01` display gap.

| Contract | Requirement |
|----------|-------------|
| Registry label source | Entity-domain Neo4j counts must use registry `neo4j_label` values from `mdm_metrics["registry"]["entity_type_details"]` or equivalent structured registry payload |
| Forbidden mapping | Do not derive Neo4j labels by stripping plural suffixes from display labels such as `Companies`, `Securities`, or `People` |
| Display label | Keep operator-facing MDM labels readable, such as `Companies`, `Advisers`, `Persons`, `Securities`, and `Funds` |
| Count lookup | Look up Neo4j node counts by exact registry label, such as `Company`, `Adviser`, `Person`, `Security`, and `Fund` |
| Regression test | Convert the expected-failure architecture regression for registry labels into a passing test |

When Neo4j is unavailable, the entity-domain coverage table should show MDM counts and `-` for Neo4j counts with status `Unavailable`; it must not show misleading zeroes.

## Interaction Contract

Data loading:

| Contract | Requirement |
|----------|-------------|
| MDM startup | MDM configuration is required. If required MDM configuration is absent, render the blocking setup page and do not continue metric loading. |
| MDM connection failure | Render the blocking MDM error state and do not render stale or partial MDM metrics. |
| Neo4j failure | Non-blocking. MDM Overview remains usable; Neo4j Overview and Mismatch Diagnostics show unavailable states for graph-backed portions. |
| Cache | Cache read-only result sets for 60 seconds. `Refresh metrics` clears cache and reruns. |
| Query safety | Streamlit code must call read-only helper functions only; it must not include raw SQL, raw Cypher, sync orchestration, migration calls, or mutation helper imports. |

Controls:

| Control | Contract |
|---------|----------|
| Refresh metrics | The only action button. Clears cached metrics and reruns. |
| Navigation | Sidebar single-choice navigation with the four labels defined above. |
| Filters | Single-select controls only, visible near the top of the relevant page. |
| Row limit | Global sidebar selectbox; no arbitrary numeric input. |

Forbidden UI:

| Forbidden | Reason |
|-----------|--------|
| Sync graph, derive relationships, load relationships, migrate, seed, merge, repair, quarantine, accept/reject, edit, delete, or credential controls | Dashboard is read-only |
| Buttons that run existing MDM/graph commands | Operator commands belong in documentation, not dashboard controls |
| Raw database URLs, Neo4j URIs, usernames, passwords, secret JSON, or driver exception text | Secret-safe UX requirement |
| Managed deployment, Terraform, Step Functions, Snowflake, dbt, generated application JSON, or rollout controls | Out of scope for this workstream phase |
