<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| committed git evidence -> shared/audit readers | Evidence files are committed and read by operators and future runtimes; they must never carry credential values. |
| deployed prod config -> committed evidence claims | The audit asserts deployment state from committed evidence; a claim that outruns its evidence (e.g. Blocker 4 concurrency) is a tampering/integrity risk. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-11-01 | Information disclosure | committed evidence .md files | mitigate | Task 2 credential-value scan (SEC-02) gates on real DSNs/access-keys/private-keys/JWTs; zero-match required, BLOCKING otherwise. Account-id/ARN recorded informational per project convention. |
| T-11-02 | Tampering | Blocker 4 status vs deployed MaxConcurrency | mitigate | Task 1 reconciles deployed concurrency against committed evidence; mismatch forced to CONDITIONAL + Open Items, never silent PASS. |

| T-11-SC | Tampering | npm/pip/cargo installs | accept | No package-manager installs in this documentation/audit phase; no install tasks exist. |
</threat_model>
