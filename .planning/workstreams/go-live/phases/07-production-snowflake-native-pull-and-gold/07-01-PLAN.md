## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator CLI -> Terraform/AWS/Snowflake | Wrapper applies production IAM trust and Snowflake infrastructure. |
| raw validation artifact -> committed evidence | Helper output can contain sensitive infrastructure identifiers. |
| generated app manifest/Terraform outputs -> Snowflake deploy inputs | Multiple value sources must agree before trust reconciliation proceeds. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Information Disclosure | `prod_native_pull_handshake.json` and Terraform outputs | mitigate | Commit only sanitized JSON and prose summaries; forbid ARNs, external IDs, S3 URLs, manifest names, account identifiers, raw rows, and raw errors. |
| T-07-02 | Tampering | AWS/Snowflake trust reconciliation | mitigate | Preflight value-source consistency; stop on manifest/Terraform/live mismatch unless only stale app manifest is contradicted by live/Terraform agreement. |
| T-07-03 | Elevation | Snowflake access-control apply | mitigate | Use existing Terraform access root and summarize role names/grant categories only; do not broaden policies manually. |
| T-07-04 | Repudiation | Wrapper success without deep proof | mitigate | SNOW-03 PASS requires independent validation checks and sanitized artifact status, not wrapper exit alone. |
| T-07-05 | Denial of Service | Manifest task or validation failure after deploy | mitigate | Rerun validation once, then record partial-pass BLOCKED evidence with owner/remediation instead of looping or masking failure. |
| T-07-SC | Tampering | Package installs | accept | No package manager install is part of this plan; commands use existing Terraform, SnowCLI, Python, and wrapper scripts. |
</threat_model>
