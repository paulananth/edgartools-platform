## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Assuming a grep/sed secret filter matches actual output shape | `CREATE POSTGRES INSTANCE`'s default TABLE format wrapped credential strings character-by-character across rows, and used JSON keys (`snowflake_admin`, `application`) the existing filter didn't cover ‚Äî leaked into chat | blocking | Added `--format JSON` to keep values unbroken; extended `redact_text()` with a pattern for those key names (commit `509ac16`) |
| Same root cause, different command | `ALTER POSTGRES INSTANCE ... RESET ACCESS FOR` returns a generic column literally named `password` ‚Äî not `snowflake_admin`/`application` ‚Äî so the first redaction fix didn't cover it; leaked again on the first rotation attempt | blocking | Added a generic `password` key pattern to `redact_text()`; made `execute_stage()` redact ALL stage output by default as defense in depth (commit `10be2d9`) |
| Secret printed via cat/grep (earlier session) | `cat connections.toml \| grep -A6` showed the literal ANANP11 password in a tool result | blocking | Always pipe credential-file reads through a redacting filter before the first read, never after |
| Assuming `snow sql -D` Jinja templating just works | `{{ var }}` syntax raised a literal SQL syntax error | advisory | `snow sql` defaults to `--enable-templating LEGACY`; pass `--enable-templating JINJA` explicitly |
| Trusting a prior session's "separate Snowflake accounts" conclusion without re-verifying | A prior HANDOFF.json decision claimed Snowflake prod/dev are separate accounts (based on differing connection-string formats) | advisory | `SELECT CURRENT_ACCOUNT()`/`CURRENT_ACCOUNT_NAME()`/`CURRENT_ORGANIZATION_NAME()` returned IDENTICAL values via both connections ‚Äî corrected this session |

**Rule going forward:** capture a command's real output shape (or read its docs/`--help` for
field names) before relying on any redaction filter to mask its output.

## decisions_made

  prod MDM instances/policies share one account-level blast radius.

  default `LEGACY` mode silently fails to substitute and raises a syntax error instead.

  assumed from a generic key-name pattern ‚Äî two incidents this session both stemmed from this.
