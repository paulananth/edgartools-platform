2. One `### ` section per system, for ALL 8 OPS-02 systems in this order:
   (1) Step Functions status, (2) CloudWatch logs, (3) Snowflake task history,
   (4) dbt test failures, (5) MDM counts, (6) hosted graph verification,
   (7) Native App compute pool health, (8) dashboard availability. Each section
   MUST contain these four labeled parts:
   - **Diagnostic** — 1-2 read-only commands in a fenced bash block.
   - **Expected output shape** — what a healthy result looks like (counts,
     statuses, freshness) without pasting real values.
   - **Escalation threshold** — the condition that triggers escalation.
   - **Owner** — the escalation owner, mirroring the launch gate matrix Owner
     column.

   Per-system diagnostics:
   - Step Functions status: `aws stepfunctions list-executions --state-machine-arn <arn> --query ...`
     and/or `describe-execution --query status`. Owner: AWS operator.
   - CloudWatch logs: `aws logs tail <log-group> --since 1h` (read-only tail).
     Owner: AWS operator.
   - Snowflake task history:
     `snow sql --connection <conn> -q "SELECT * FROM TABLE(<DB>.INFORMATION_SCHEMA.TASK_HISTORY()) ... LIMIT 10;"`
     and/or `SHOW TASKS LIKE 'SNOWFLAKE_RUN_MANIFEST_TASK';`. Owner: Snowflake operator.
   - dbt test failures: `uv run --with dbt-snowflake dbt test --target prod`
     (reports pass/fail; assertions are read-only). Owner: Snowflake operator.
   - MDM counts: `edgar-warehouse mdm counts` (read-only). Owner: MDM operator.
   - hosted graph verification: `edgar-warehouse mdm verify-graph` (read-only
     parity + Native App checks). Owner: MDM operator.
   - Native App compute pool health:
     `snow sql --connection <conn> -q "SHOW COMPUTE POOLS;"` (read-only state).
     Owner: Snowflake operator.
   - dashboard availability: `pgrep -f streamlit` and/or a read-only Streamlit
     health-check URL probe. Owner: dashboard reviewer.

3. Cross-reference (short): state that escalation owners and per-layer routing
   mirror the launch gate matrix Data-Issue Triage Table; link it via the
   cross-phase relative path
   `../01-production-readiness-inventory-and-launch-gate-contract/01-LAUNCH-GATE-MATRIX.md`.
   This checklist EXTENDS that table with post-launch monitoring cadence — it
   does not duplicate the blocker columns.
