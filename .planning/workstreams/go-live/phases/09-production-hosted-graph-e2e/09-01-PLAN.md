## Trust Boundaries

| Boundary | Description |

| AWS Secrets Manager -> operator shell | Existing prod MDM secrets are loaded into env vars and must never be printed. |
| operator shell -> production Snowflake | Native App grants, graph tables, and graph algorithm output tables are production state changes. |
| MDM CLI -> evidence Markdown | Only sanitized summaries cross into git. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|---|---|---|---|---|
| T-09-01 | Information Disclosure | Existing prod secret values printed during graph sync | mitigate | Load secrets only inside one non-printing shell invocation; no `set -x`; unset values before exit; grep evidence for known leak patterns. |
| T-09-02 | Tampering | Dev Native App grant script run unchanged against prod evidence | mitigate | Task 3 requires production-scoped target `EDGARTOOLS_PROD.NEO4J_GRAPH_MIGRATION`; evidence must state prod target. |
| T-09-03 | Denial of Service / Cost | Unbounded MDM or Native App graph run | mitigate | Require explicit limits: `mdm run --limit 5`, `backfill-relationships --limit 100`, `sync-graph --limit 100`; operator approval checkpoint. |
| T-09-04 | Repudiation | Matrix row flipped PASS before verify-graph | mitigate | No matrix edit in 09-01; PASS reconciliation is deferred to 09-02 after AWS E2E. |
| T-09-SC | Tampering | package install surface | accept | No new package dependencies are planned; existing `uv` extras only. |
</threat_model>
