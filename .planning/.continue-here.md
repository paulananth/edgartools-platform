# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: NEVER-PRINT-CREDENTIALS — `/tmp/mdm-cutover-secrets/` contains live plaintext credentials (DSN files, JSON secrets). Never echo, cat, or print these file contents — always mask passwords (e.g. `user:***@host`) before displaying. Never embed real passwords into any script, doc, or commit.
- [ ] CONSTRAINT: DEPLOY-SCRIPT-STALE-ARN — `deploy-aws-application.sh` prefers a CACHED manifest ARN over a live Secrets Manager lookup for `EDGAR_IDENTITY`. If the secret was recreated since the last deploy (ARN suffix changes), this silently deploys broken task defs. **Always pass `--edgar-identity-secret-arn $(aws secretsmanager describe-secret --secret-id edgartools-dev-edgar-identity --region us-east-1 --query ARN --output text)` explicitly.**
- [ ] CONSTRAINT: DEPLOY-SCRIPT-ECR-CLEANUP — `deploy-aws-application.sh`'s own ECR cleanup step deletes images that don't match the `:sha-<hash>` tag pattern — including the digest you just resolved via `describe-images[-1]`. **Always re-resolve ECR digests fresh immediately before each deploy run.** Never reuse a digest from before a prior deploy run.
- [ ] CONSTRAINT: MDM-CHECK-CONNECTIVITY-SFN-BROKEN — The `edgartools-dev-mdm-check-connectivity` Step Functions state machine hardcodes `--neo4j` which was removed from the CLI in commit `8784fd5`. **Do NOT use this state machine.** Run connectivity checks via direct `ecs run-task` with command `["mdm","check-connectivity"]` (no flags).
- [ ] CONSTRAINT: STEP-6-IRREVERSIBLE — Step 6 (RDS removal via `remove-aws-mdm-rds-after-cutover.sh`) is **irreversible** and must be preceded by an explicit user checkpoint. Do NOT run it until: (a) `coverage-report` shows full parity, (b) audit gate (Step 5) passes, AND (c) user explicitly approves.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Stale secret ARN in manifest | deploy-aws-application.sh cached a dead edgar-identity ARN (suffix `UMvCTb`) from pre-rotation manifest, deployed broken task defs silently | blocking | Always pass `--edgar-identity-secret-arn` with freshly-looked-up ARN |
| ECR self-deletion | Deploy script's cleanup deleted the very image digest the same run was deploying (tag `0bb535b` ≠ expected `sha-*` pattern) | blocking | Re-resolve digests fresh immediately before each deploy invocation |
| State machine CLI drift | `mdm-check-connectivity` SFN hardcodes removed `--neo4j` flag; first exercise of it post-decommission caused 3 failed deployments before root cause found | blocking | Always test new SFN invocations with direct `ecs run-task` first |
| Local batched population | Local batched mdm runs via `--limit 500` attempted first, but `--offset` without `ORDER BY` is unreliable; local rate ~534ms/row makes full run ~15h | advisory | Run all MDM population from ECS (same network, no connection-drop issue, proven stable at 49 min for 5500 companies) |
