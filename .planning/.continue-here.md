## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention |
|---|---|---|---|
| Heredoc commit/PR messages | `git commit -m "$(cat <<'EOF' ... EOF)"` and `gh pr create --body "$(cat <<'EOF' ...)"` both threw `unexpected EOF while looking for matching` in this shell. | advisory | Write the message to a `/tmp/*.txt` or `/tmp/*.md` file with the Write tool, then `git commit -F <file>` / `gh pr create --body-file <file>`. |
| `find` over repo root | `find . -iname "*foo*"` occasionally hit `ENOSPC: temp filesystem ... 0MB free` (transient — disk actually had 8.6GB free). | advisory | If it happens, retry with a narrower path (e.g. `find tests -iname ...`) or via `grep -r`. |
| `gh pr merge --delete-branch` deletes local branch + unpushed commits | After PR #56 merged, the local feature branch (with an unpushed WIP handoff commit `38e2ab8`) was deleted by `--delete-branch`, orphaning that commit. | advisory | Recovered via `git reflog` + `git fsck --unreachable` + `git show <sha>:<path>`. Push WIP commits before merging the PR, or omit `--delete-branch` until local state is checked. |

<decisions_made>

- Two-pass staging-table UPSERT (first-seq `ON CONFLICT DO NOTHING` +
  last-seq `ON CONFLICT DO UPDATE SET <mutable cols>`) chosen over a
  single-QUALIFY approach because the old row-by-row loop has per-column
  "first-insert-wins vs last-write-wins" semantics that a single dedup pass
  cannot replicate. Validated via differential test.
- `backfill_accounting_flags` fix relies on `update_accounting_flag_scores`'s
  existing `COALESCE(?, existing_column)` SQL — no new fallback-fetch query
  needed.
</decisions_made>
